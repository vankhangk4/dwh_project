🔷 PHASE 0: Project Scaffolding

  - PROJECT_ROOT/setup.py — pyproject.toml với dependencies (pandas, pyodbc, openpyxl, fastapi, uvicorn, pyjwt, passlib[bcrypt],
  python-dotenv, schedule)
  - docker-compose.yml — Services: sqlserver, superset, etl-scheduler, api (FastAPI)
  - .env.example — Template env vars (JWT_SECRET, CONN_STR, SMTP_*, SUPERSET_*, SLACK_*)
  - README.md — Hướng dẫn chạy, cấu trúc thư mục
  - requirements.txt

  🔷 PHASE 1: SQL Schema — Tenant Core (sql/schema/01_create_tenants.sql)

  - Bảng Tenants (TenantID PK, TenantName, FilePath, IsActive, CreatedAt)
  - Bảng AppUsers (UserID PK, Username UNIQUE, PasswordHash bcrypt, TenantID FK nullable, Role CHECK('admin'/'viewer'), IsActive,
  CreatedAt)
  - Insert seed: STORE_HN, STORE_HCM

  ---
  🔷 PHASE 2: SQL Schema — Dimensions Shared (sql/schema/02_create_dimensions.sql)

  - DimDate — Shared, pre-populated 2015→2030 (DateKey INT yyyyMMdd, FullDate, Year, Month, Day, Quarter, DayOfWeek, IsHoliday)
  - DimProduct — Shared, SCD Type 2 (ProductKey PK, ProductCode, ProductName, Brand, CategoryName, UnitCostPrice, UnitListPrice,
  IsActive, EffectiveDate, ExpirationDate, IsCurrent)
  - DimSupplier — Shared (SupplierKey PK, SupplierCode, SupplierName, ContactName, Phone, Email, City, Country, IsActive)

  ---
  🔷 PHASE 3: SQL Schema — Dimensions Tenant-Specific (sql/schema/03_create_dimensions_tenant.sql)

  - DimStore — Có TenantID (StoreKey PK, TenantID, StoreCode, StoreName, Address, City, District, Phone, ManagerName, OpenDate,
  IsActive)
  - DimCustomer — Có TenantID, SCD Type 2 (CustomerKey PK, TenantID, CustomerCode, FullName, Gender, City, CustomerType, LoyaltyPoint,
  MemberSince, IsActive, EffectiveDate, ExpirationDate, IsCurrent)
  - DimEmployee — Có TenantID (EmployeeKey PK, TenantID, EmployeeCode, FullName, Gender, DateOfBirth, Position, Department, HireDate,
  Phone, Email, IsActive)

  ---
  🔷 PHASE 4: SQL Schema — Facts (sql/schema/04_create_facts.sql)

  - FactSales — Grain: 1 dòng = 1 sản phẩm/hóa đơn. Có TenantID. Columns: TenantID, DateKey, ProductKey, CustomerKey, StoreKey,
  EmployeeKey, InvoiceNumber, Quantity, UnitPrice, DiscountAmount, GrossSalesAmount, NetSalesAmount, CostAmount, GrossProfitAmount,
  PaymentMethod, SalesChannel, ReturnFlag, LoadDatetime
  - FactInventory — Grain: 1 sản phẩm/cửa hàng/ngày. Có TenantID. Columns: TenantID, DateKey, ProductKey, StoreKey, OpeningQty,
  ReceivedQty, SoldQty, AdjustedQty, ClosingQty, StockValue, ReorderLevel, LoadDatetime
  - FactPurchase — Grain: 1 dòng nhập hàng. Có TenantID. Columns: TenantID, DateKey, ProductKey, SupplierKey, StoreKey,
  PurchaseOrderNumber, Quantity, UnitCost, TotalCost, PaymentStatus, LoadDatetime

  ---
  🔷 PHASE 5: SQL Schema — Staging Layer (sql/schema/05_create_staging.sql)

  - STG_SalesRaw — (Có TenantID) — 1 row per product per invoice
  - STG_InventoryRaw — (Có TenantID) — 1 row per product per store per day
  - STG_ProductRaw — (Shared) — Danh mục sản phẩm
  - STG_CustomerRaw — (Có TenantID)
  - STG_EmployeeRaw — (Có TenantID)
  - STG_PurchaseRaw — (Có TenantID)
  - ETL_Watermark — (Có TenantID) — SourceName, TenantID, WatermarkValue, LastRunStatus, LastRunDatetime, RowsExtracted, Notes
  - STG_ErrorLog — (Có TenantID) — TenantID, SourceTable, ErrorType, ErrorMessage, RawData, LoadDatetime

  🔷 PHASE 10: ETL — Transform Module (etl/transform/)

  - __init__.py
  - transform_sales.py — Full logic: chuẩn hóa datetime/int/float, trim/UPPER strings, xử lý NULL, loại trùng (MaHoaDon+MaSP), filter

  - transform_customer.py — Chuẩn hóa KH, xử lý loyalty point, ngày đăng ký
  - transform_employee.py — Chuẩn hóa nhân viên, tính tuổi, thâm niên
  - transform_purchase.py — Chuẩn hóa nhập hàng, tính TotalCost
  - base_transform.py — Shared logic: clean_string(), parse_date(), handle_null()
